# 02 Puzzle Mechanics

## 문서 메타
- 목적: 퍼즐 핵심 규칙(매칭, 보드 상태, 승패 판정)을 정의한다.
- 소유자: 기획 + 클라이언트
- 상태: Draft
- 관련 문서: [01-product-core](./01-product-core.md), [03-content-level-balance](./03-content-level-balance.md)

## 보드 모델
- 그리드: `TBD` (예: 8×8)
- 타일 종류: 기본 이모지 N종 + 장애물 + 특수 타일
- 좌표 체계: `(col, row)`, 원점 좌상단
- 상태: `tile | empty | obstacle | locked`

## 매칭 규칙
- 기본: 동일 타일 3개 이상이 가로 또는 세로로 연속
- 4매치: 라인 폭발(같은 줄 일괄 제거) 특수 타일 생성
- 5매치: 전체 색 제거형 특수 타일 생성
- T/L 매치: 주변 영역 폭발 특수 타일 생성
- 연쇄(cascade): 제거 후 낙하로 새로 형성된 매치는 자동으로 다시 해소

## 턴 흐름
1. 플레이어 입력(인접 스왑)
2. 매치 가능 여부 검사 — 불가능이면 스왑 롤백
3. 매치 해소 → 점수/연쇄 카운트 증가
4. 낙하 및 빈칸 충전
5. 추가 매치 자동 해소 반복 (cascade)
6. 보드가 정지하면 턴 종료, 목표 판정

## 승패 판정
- 승리: 목표 달성 시점(점수·수집·제거 완료)에 즉시 WIN
- 패배: 제한(이동수 또는 시간) 소진 + 목표 미달성
- 보드 경직: 가능한 수가 없을 경우 자동 셔플

## 불변식(Invariants)
- 보드 초기 생성 시 즉시 매치가 없어야 한다.
- 보드 초기 생성 시 최소 1수 이상의 유효 수가 존재해야 한다.
- cascade 중에는 플레이어 입력이 무시된다.
- 특수 타일 효과는 트리거 후 동일 턴 내에 모두 해소된다.

## 오픈 이슈
- 특수 타일 결합 시 효과(e.g. 라인+라인, 라인+컬러) 규칙 확정
- 장애물 계층화 (얼음/자물쇠/숨김) 범위
- 셔플 빈도 제한(UX) 및 시각 피드백

# 04 Emoji Art / UI

## 문서 메타
- 목적: 이모지 매지션 퍼즐 콘셉트의 시각 규칙과 UI 우선순위를 정의한다.
- 소유자: 기획 + UI
- 상태: Draft
- 관련 문서: [01-product-core](./01-product-core.md), [02-puzzle-mechanics](./02-puzzle-mechanics.md)

## 아트 방향
- 모든 퍼즐 타일을 이모지로 표현한다.
- 배경은 단순화하고 보드 가독성을 최우선으로 한다.
- 타일 간 대비(모양/색 차이)를 명확하게 구분한다.

## 이모지 매핑 초기안
- 기본 타일: `🍎`, `🍋`, `🍇`, `🍒`, `🥝`, `🥥` (6종 중 5~6개 사용)
- 특수 타일: `💥`(라인), `🌈`(컬러), `🎇`(범위)
- 장애물: `🧊`(얼음), `🔒`(자물쇠), `🧱`(벽)
- UI 리소스: `⭐`(별), `💎`(보석), `🪙`(코인), `❤️`(하트)

## HUD 우선순위
1. 레벨 번호, 목표(아이콘+수치), 남은 이동/시간
2. 현재 점수 및 별 달성 기준선
3. 사용 가능한 부스터

## 입력 UI
- 인접 타일 간 탭-탭 또는 드래그 스왑
- 불가능한 스왑은 짧은 shake 피드백 후 원복
- 특수 타일은 탭 한 번으로 발동 가능 (설계에 따라)

## 애니메이션 규칙
- 스왑: 200ms ease
- 제거: 150ms fade + pop
- 낙하: 중력감 있는 ease-in
- 연쇄: 동시 애니메이션 큐로 묶어 처리

## 오픈 이슈
- 플랫폼별 이모지 렌더링 차이 대응(폰트 fallback) 전략
- 색약/색맹 접근성 모드 (타일 심볼 오버레이)
- 가로/세로 모드 지원 범위

## 1. 게임 개요

### 1-1. 기본 정보

| 항목 | 내용 |
|------|------|
| 게임 ID | G1 |
| 게임명 | 수박 터뜨리기 (Watermelon Burst) |
| 타깃 | 초등 1학년 (7~8세) |
| 세션 길이 | 60초 / 레벨 (3레벨 구성, 총 최대 3분) |
| 학습 목표 | 합이 10이 되는 두 수 인식 (덧셈 보수, 수학 교육 커리큘럼) |
| 테마 | 수박 🍉 — 여름, 달콤함, 터지는 쾌감 |
| 핵심 감정 | 고름의 긴장 → 탭 → 터짐의 쾌감 → 반복 숙달 |

### 1-2. 학습 목표 상세

합이 10이 되는 쌍(보수 쌍)은 초등 1학년 수학의 핵심 개념이다.
- 1+9, 2+8, 3+7, 4+6, 5+5 (총 5가지 쌍 유형)
- 게임 반복을 통해 보수 쌍을 암산이 아닌 패턴 인식 수준으로 내면화하는 것이 목표이다.

### 1-3. 플레이어 경험 목표
1. 규칙을 설명 없이 첫 30초 안에 스스로 터득한다 (자명한 인터랙션).
2. 터질 때마다 즉각적 쾌감 피드백이 행동을 강화한다.
3. 레벨 3 클리어 후 "나 이제 10 만들기 잘한다"는 성취감을 느낀다.

## 2. 보드 / 타일 스펙

### 2-1. 보드

| 항목 | 값 |
|------|----|
| 그리드 | 4열 x 5행 = 20칸 |
| 좌표 체계 | (col, row), 원점 좌상단 (0,0) |
| 타일 상태 | `filled` / `empty` / `selected` / `exploding` |
| 빈 칸 처리 | 타일이 터진 자리는 빈 칸으로 남는다 (낙하/리필 없음) |

> 설계 의도: 낙하/리필을 제거하면 보드 상태 변화가 단순해져 7세 아동이 혼란 없이 인과관계를 파악할 수 있다.

### 2-2. 타일 종류

모든 타일은 수박 조각 이모지 위에 숫자를 표시한다.

| 숫자 | 표현 이모지 | 비고 |
|------|------------|------|
| 1 | 🍉 + **1** | 얇은 조각 |
| 2 | 🍉 + **2** | |
| 3 | 🍉 + **3** | |
| 4 | 🍉 + **4** | |
| 5 | 🍉 + **5** | 중앙 조각 |
| 6 | 🍉 + **6** | |
| 7 | 🍉 + **7** | |
| 8 | 🍉 + **8** | |
| 9 | 🍉 + **9** | 굵은 조각 |

> 구현 방식: Phaser Text 오브젝트를 수박 이모지 배경 위에 오버레이한다. 숫자 폰트는 굵고 크게 (최소 32px), 배경과 대비가 강한 흰색+검은 외곽선을 사용한다.

### 2-3. 숫자 분포 원칙

**레벨 공통 불변식**
- 보드 20칸에 배치된 숫자들은 합이 10인 쌍이 정확히 10쌍 존재해야 한다.
- 20칸 전부 사용: 10개의 쌍 = 20개 타일 = 보드 전체를 채운다.

**레벨별 숫자 구성**

| 레벨 | 허용 숫자 범위 | 쌍 구성 특징 |
|------|--------------|-------------|
| 1 (쉬움) | 1, 4, 5, 6, 9 | 시각적으로 차이가 큰 숫자만 사용, 쌍이 인접 배치 |
| 2 (보통) | 1~9 전체 | 무작위 분포, 쌍이 인접하지 않을 수 있음 |
| 3 (어려움) | 1~9 전체 | 혼란 유도 배치 (동일 숫자 인접, 트랩 쌍 존재) |

### 2-4. 초기 배치 조건

1. 합이 10이 되는 쌍이 정확히 10쌍 존재해야 한다.
2. 한 숫자가 여러 쌍과 매칭될 수 있는 경우 (예: 5가 두 개) 해결 경로가 1가지 이상이어야 한다.
3. 레벨 1에서는 최소 3쌍이 인접 배치(터치 한 번에 바로 발견 가능)되어야 한다.
4. 동일 숫자가 3개 이상 연속 인접하는 배치는 레벨 1~2에서 금지한다 (혼란 방지).

## 3. 핵심 인터랙션

### 3-1. 선택 → 판정 → 결과 흐름

```
[IDLE]
  |
  | 타일 첫 번째 탭
  v
[FIRST_SELECTED]
  - 선택된 타일: 강조 (scale up + glow)
  - HUD 힌트: "다른 수박을 골라봐요!"
  |
  | 같은 타일 재탭 (선택 취소)
  +-------> [IDLE]
  |
  | 다른 타일 두 번째 탭
  v
[JUDGING]  (판정 중, 입력 차단)
  |
  | sum == 10
  +-------> [EXPLODE]
  |           - 두 타일 폭발 애니메이션 (300ms)
  |           - 점수 팝업 (+100 등)
  |           - 칸이 empty 상태로 전환
  |           - 콤보 카운터 +1
  |           - 목표 달성 여부 체크
  |               | 10쌍 달성
  |               +---> [LEVEL_CLEAR]
  |               | 미달성
  |               +---> [IDLE]
  |
  | sum != 10
  +-------> [WRONG]
              - 두 타일 shake 애니메이션 (200ms)
              - 선택 상태 초기화
              - 오류 사운드 큐
              v
           [IDLE]
```

### 3-2. 시간 소진 흐름

```
[IN_GAME] 타이머 0초 도달
  |
  | 목표 달성 (10쌍 완료)
  +---> [LEVEL_CLEAR]  (타임아웃과 클리어 동시 발생 시 클리어 우선)
  |
  | 목표 미달성
  +---> [TIME_OVER]
          - 결과 화면 (fail 상태)
          - 재도전 버튼 활성화
```

### 3-3. 입력 차단 규칙

- `JUDGING`, `EXPLODE`, `WRONG`, `LEVEL_CLEAR`, `TIME_OVER` 상태에서는 모든 타일 탭 입력을 무시한다.
- 상태 전환은 애니메이션 완료 콜백 이후에만 발생한다.

## 4. 레벨 테이블

### 4-1. 레벨 개요

| 레벨 | 이름 | 제한 시간 | 목표 쌍 수 | 난이도 핵심 |
|------|------|-----------|------------|------------|
| 1 | 달콤한 시작 | 60초 | 10쌍 | 큰 숫자 차이, 인접 쌍 다수 |
| 2 | 수박 한 판 | 60초 | 10쌍 | 전체 숫자 범위, 무작위 배치 |
| 3 | 씨 없는 수박 | 60초 | 10쌍 | 혼란 유도 배치, 속도 압박 |

### 4-2. 레벨 1 — 달콤한 시작

| 항목 | 값 |
|------|----|
| 사용 숫자 | 1, 4, 5, 6, 9 |
| 쌍 구성 | (1+9)×3, (4+6)×3, (5+5)×2, (1+9)×1, (4+6)×1 = 10쌍 |
| 인접 쌍 보장 | 최소 5쌍이 서로 인접 배치 |
| 별점 기준 | 3★: 45초 이내 / 2★: 60초 이내 / 1★: 재도전 없이 클리어 |

### 4-3. 레벨 2 — 수박 한 판

| 항목 | 값 |
|------|----|
| 사용 숫자 | 1~9 전체 |
| 쌍 구성 | (1+9)×2, (2+8)×2, (3+7)×2, (4+6)×2, (5+5)×2 = 10쌍 |
| 인접 쌍 보장 | 최소 2쌍만 인접 |
| 별점 기준 | 3★: 40초 이내 / 2★: 60초 이내 / 1★: 클리어 |

### 4-4. 레벨 3 — 씨 없는 수박

| 항목 | 값 |
|------|----|
| 사용 숫자 | 1~9 전체 |
| 쌍 구성 | (1+9)×2, (2+8)×2, (3+7)×2, (4+6)×2, (5+5)×2 = 10쌍 |
| 혼란 요소 | 동일 숫자 인접 배치 (예: 3, 3이 인접 — 오답 유도) |
| 인접 쌍 보장 | 0쌍 (모든 쌍이 비인접) |
| 별점 기준 | 3★: 35초 이내 / 2★: 60초 이내 / 1★: 클리어 |

## 5. 점수 시스템

### 5-1. 기본 점수

| 이벤트 | 점수 |
|--------|------|
| 쌍 터뜨리기 1회 | +100점 |
| 콤보 2연속 | +50 보너스 |
| 콤보 3연속 | +100 보너스 |
| 콤보 4연속 이상 | +150 보너스 (이후 고정) |
| 오답 탭 패널티 | 없음 (7세 타깃) |
| 시간 내 클리어 잔여 시간 보너스 | 잔여초 × 10점 |

### 5-2. 별점 산출

| 별점 | 기준 |
|------|------|
| 3★ | 여유 있게 클리어 (레벨별 시간 기준) |
| 2★ | 제한 시간 내 클리어 |
| 1★ | 재도전 없이 클리어 (1회 플레이) |
| 0★ | 시간 초과 후 미클리어 |

### 5-3. 점수 팝업 연출

- 기본 점수 (+100): 터진 두 타일 중심에서 위로 float + fade (500ms)
- 콤보 보너스: 화면 중앙 상단 큰 텍스트 팝업 (700ms)
- 폰트 크기: 기본 36px / 콤보 54px
- 색상: 기본 흰색 / 콤보 노란색 (#FFD700)

## 8. 기술 노트

### 8-1. DOM vs Phaser 역할 분리

| 레이어 | 기술 | 담당 범위 |
|--------|------|----------|
| 메인 메뉴 | DOM | 타이틀, 레벨 선택, 해금 상태 |
| 레벨 인트로 오버레이 | DOM | 목표 안내, 카운트다운 |
| HUD | DOM | 남은 시간, 점수, 레벨, 남은 쌍 수 |
| 게임 보드 + 타일 | Phaser | 4x5 그리드, 입력, 애니메이션, 이펙트 |
| 결과 화면 | DOM | 별점, 점수, 메시지, 버튼 |

DOM HUD는 Phaser 캔버스 위에 `position: absolute`로 오버레이한다.

### 8-2. 핵심 시스템 인터페이스

```typescript
// src/game-data/g1-levels.ts
interface G1LevelData {
  levelId: number;
  timeLimit: number;        // 초 단위 (60)
  targetPairs: number;      // 목표 쌍 수 (10)
  allowedNumbers: number[]; // 사용 가능 숫자
  adjacentPairGuarantee: number; // 인접 쌍 최소 보장 수
  starThresholds: { star3: number; star2: number };
}

// src/systems/g1BoardGenerator.ts — 순수 함수
function generateBoard(levelData: G1LevelData): number[][];

// src/systems/g1PairJudge.ts — 순수 함수
function judgePair(a: number, b: number): 'correct' | 'wrong';
```

### 8-3. 우선 테스트 목록

| 테스트 대상 | 검증 내용 |
